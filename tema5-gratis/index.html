5.1 Tipos de redes
LAN, MAN, WAN, PAN — clasificación por extensión geográfica
TipoSignificadoExtensiónEjemplo
PANPersonal Area NetworkPocos metrosBluetooth entre móvil y auriculares
LANLocal Area NetworkEdificio/campusRed de una oficina
MANMetropolitan Area NetworkCiudadRed universitaria de una ciudad
WANWide Area NetworkPaís/continente/mundoInternet
Topología de red: la forma en que se conectan los dispositivos. Principales: bus (un cable central), estrella (todos conectados a un switch central — la más común en LAN), anillo, malla.
La topología estrella es la más usada en redes locales modernas. Si el switch central falla, toda la red cae; pero si falla un equipo, el resto sigue funcionando.
WLAN (Wireless LAN): red local inalámbrica. La tecnología estándar es Wi-Fi (IEEE 802.11). Las versiones más recientes son Wi-Fi 6 (802.11ax) y Wi-Fi 7 (802.11be).
📡
5.2 Protocolos TCP/IP e IP
Dirección IP, máscara de subred, puertos
El protocolo TCP/IP es la base de Internet. TCP (Transmission Control Protocol) garantiza la entrega de los datos. IP (Internet Protocol) se encarga del direccionamiento.
Una dirección IP es un identificador único para cada dispositivo en la red. IPv4: 4 números del 0-255 separados por puntos (ej. 192.168.1.100). IPv6: 128 bits en hexadecimal.
La máscara de subred divide la dirección IP en parte de red y parte de host. Ejemplo: 255.255.255.0 o /24 (notación CIDR).
Tipos de IP: pública (visible en Internet, única globalmente) y privada (solo válida en redes locales). Rangos privados: 192.168.x.x, 10.x.x.x, 172.16-31.x.x.
IP estática: asignada manualmente, siempre la misma. IP dinámica: asignada automáticamente por un servidor DHCP cada vez que el dispositivo se conecta.
Puertos: identifican el servicio específico. Ejemplos: HTTP = 80, HTTPS = 443, FTP = 21, SSH = 22, SMTP = 25, DNS = 53. Los puertos van del 0 al 65.535.
🔍
5.3 DNS y servicios de Internet
DNS, DHCP, HTTP, FTP, correo electrónico
El DNS (Domain Name System) traduce los nombres de dominio (www.google.com) a direcciones IP numéricas. Sin DNS, habría que memorizar las IPs de cada web.
El DHCP (Dynamic Host Configuration Protocol) asigna automáticamente direcciones IP, máscara de subred, puerta de enlace y DNS a los dispositivos cuando se conectan a la red.
HTTP (HyperText Transfer Protocol): protocolo para la web. Puerto 80. Sin cifrado — los datos viajan en texto plano. HTTPS: versión segura con SSL/TLS. Puerto 443.
FTP (File Transfer Protocol): transferencia de archivos. Puerto 21 (control) y 20 (datos). SFTP/FTPS: versiones seguras cifradas.
Correo electrónico: protocolos de envío: SMTP (puerto 25/587). Protocolos de recepción: POP3 (puerto 110, descarga el correo) e IMAP (puerto 143, sincroniza sin descargar).
Una URL (Uniform Resource Locator) tiene la estructura: protocolo://dominio:puerto/ruta?parámetros. Ejemplo: https://www.ejemplo.com:443/pagina?id=1
🌐
5.4 Navegadores web
Principales navegadores, caché, cookies, marcadores
Navegadores principales: Google Chrome (más usado mundialmente), Mozilla Firefox (código abierto), Microsoft Edge (integrado en Windows), Safari (Apple), Opera.
La caché del navegador almacena copias de páginas web visitadas para cargarlas más rápido. Se puede borrar desde la configuración del navegador (Ctrl+Mayús+Supr en la mayoría).
Las cookies son pequeños archivos que los sitios web guardan en el navegador para recordar preferencias, sesiones iniciadas y datos de navegación.
El modo incógnito/privado no guarda historial, cookies ni datos de formularios. Pero NO te hace anónimo frente a tu ISP, la red o los sitios visitados.
Los marcadores/favoritos guardan accesos directos a páginas web. Se pueden organizar en carpetas y sincronizar entre dispositivos con la cuenta del navegador.
Extensiones/complementos: pequeños programas que añaden funcionalidades al navegador. Pueden ser bloqueadores de anuncios, gestores de contraseñas, traductores, etc.
🛡️
5.5 Seguridad en redes: VPN y Firewall
VPN, firewall, proxy y redes seguras
Una VPN (Virtual Private Network) crea un túnel cifrado entre el dispositivo y un servidor remoto. Cifra el tráfico y oculta la IP real. Usada por empresas para acceso remoto seguro y por usuarios para privacidad.
Un firewall (cortafuegos) filtra el tráfico de red entrante y saliente según reglas definidas. Puede ser software (Windows Firewall) o hardware (dispositivo físico dedicado).
Un proxy actúa como intermediario entre el cliente y el servidor web. Puede cachear contenido (proxy caché), filtrar accesos (proxy de contenido) o proporcionar anonimato.
WPA3 es el estándar de seguridad Wi-Fi más actual. Versiones anteriores: WEP (obsoleto, inseguro), WPA, WPA2. Para redes domésticas se recomienda WPA2 o WPA3.
HTTPS usa certificados SSL/TLS para cifrar la comunicación entre el navegador y el servidor web. El candado verde/gris en la barra del navegador indica conexión HTTPS.
